S3OP

 

 

 

 

 

 

A SOP designing software suit, that not only standardises but simplifies!

Summary

 

How this idea came into being.

Hi im Connor Litchfield, Auto electrician by trade and storeman by job description. During my time working for an extremely large organisation, I have noticed one glaring flaw, that is the SOP (Standard Operating Procedure) system. Nothing is standardised no two look the same, no two are stored in the same place, and there are repeats of repeats with differing information for the same outcome. But need not despair I think I have found the solution.

 

What is a SOP?

A SOP is a Standard Operating Procedure, this can take the form of a written set of instructions teaching the perspective reader how to safely perform a specific task.

The tasks can involve a high-risk physical task, i.e. mechanical repair, use of a certain piece of machinery even just a introduction to picking the right PPE for the right job based on common sense risk, cuts, abrasions, eye damage, hearing damage, heat injuries, cold injuries, crushing injuries, muscular or skeleton injuries.

 

Introducing S3OP!

S3OP the future of SOP design and standardisation.

Super Simple Standard Operating Procedures or S3OP for short, is a concept for a new software that will undo the complicated nature of creating and maintaining SOP libraries.

All while standardizing and centralising your SOPs.

A SOP is a Standard Operating Procedure, so why are no two alike? With S3OP they will be! S3OP will have a locked down template designed by your team, this means that collaborators can create to their hearts content, but every SOP in your library will look and read the same way.

A SOP is a Standard document so why is there one here and there and some in that folder. and a few on the network drive. If this is meant to be standard, then why arent these stored in a standard way! Well with S3OP they will be! S3OP has a fully customisable tree style library structure, so that any collaborator can easily find existing SOPs before creating new ones and find where to put their shiny new SOP so that its easy to find and read for those who need it most.

How it works.

The standard template

The S3OP is fully automated and will generate a standardized and complete SOP for the user with minimal input required.

S3OP will including a built-in word-processing interface for every step, this will allow the user to enter text with all the formatting they need.

This software will have an easy to use interface.

1.      The user is prompted to supply a title and photo that will be used for the SOP front cover.

2.      The software will ask the user for a step to go into their SOP and a picture if applicable to be added to this step.

3.      The user now adds as many steps as they like.

As the user enters a step to the SOP a bar along the bottom of the screen will show a new point along their SOP timeline.

4.      The user Saves the document to a position in the tree library.

5.      The user can Print the SOP if desired, this will place a mandatory

Printed on dd/mm/yyyy to show readers how old this document is.

The software has the ability for the user to delete and edit steps, all while auto saving the document as they go. This can be implemented in very simple way for the user, the user just clicks a step in the time line bar and S3OP will take the user to that step and display it in edit mode.

 

Permissions

A permission system will be implemented this. This will take the form of a hierarchy system, where the user who creates the document has implicate access to modify their new document.

A next in the hierarchy will be the next in charge, i.e. manager. The manager will have access to modify any document created by a user under their position, this position can specific if a user below their position does or does not have creation / edit permission. This allows the manager to specify specific people for their team who can modify / create documents, all while allowing any user (Guest) to view SOPs from S3OP.

This hierarchy system will continue like this until, eventually there is one user or group of users with ultimate modify permission, i.e. your Tech Support team.

 

What skills will I need to create this?

I believe the best way to accomplish this would be using HTML5. HTML5 would give the easiest to program and most visually appealing outcome.

HTML5 & CSS

So I need to have a much better understanding of HTML5 than what I currently do, and advanced skills in CSS usage is also required.

 

GUI Design

I will need to learn and understand GUI creation, so that I can design something that is not only visually appealing but useable. Without being overly complex for the user.

User permission management protocols

I will need to learn and understand, how to implement and maintain a user permission management system, so that the software has a place where it can draw on permission status for a user, before allowing Create/Edit/Modify access.

Data storage

How & where is this software going to store all these documents, I will need to research this heavily as finding a one fits all solution for every user maybe be very difficult, especially if the customer wishes to store the SOP library on their own servers.

 

What will S3OP accomplish?

S3OP will unify the creation of standard operating procedures, this will make storying creating and editing SOPs uniform and standard. But most importantly this will make finding a SOP so much easier and faster for a reader.

 

Imaging a situation where an employee comes to a new task for them, and a task with inherent risk at that. The employee makes the smart decision to seek out a SOP so they can perform this task in a smart safe way. But cannot find the SOP now mater where they search, or the employee decides not to bother searching at all, because they know that the search will take a very long time and will probably yield no results. Now this employee will just perform this task by just winging it.

 

Duplication will also be solved, if a creator looks in the SOP tree and can see a SOP that already exists for the subject matter than there is no need to create a new one, this means no duplicates.
